## Pre-Batch

Read [references/pre-batch.md](references/pre-batch.md) for the full checklist. Key actions:

1. **Incorporate** — Delaware C-Corp via Clerky or Stripe Atlas. Execute founder vesting (4yr/1yr cliff). File 83(b) elections within 30 days.
2. **Banking** — Open startup bank account (Mercury/Brex). Set up bookkeeping.
3. **Ship before you arrive** — Get an MVP live. Talk to 10+ users.
4. **Co-founder alignment** — Have explicit conversations about equity, roles, commitment, and vision.
5. **Pivot readiness** — Write down your top 3 assumptions. Pre-commit to reassessing after 2-3 weeks.

Use the pre-batch reference for detailed checklists on legal setup, SAFE notes, team alignment, and idea validation.

## Building & Shipping

Read [references/building.md](references/building.md) for the full playbook. Core principles:

- **Ship fast.** 24-48 hour cycles. Deploy daily.
- **Talk to users every day.** 5-10 conversations per week minimum.
- **Do things that don't scale.** Manual concierge for the first users.
- **Cut scope ruthlessly.** Build the worst viable product that tests your hypothesis.

The building reference includes the batch sprint cadence, the "Worst Viable Product" scoping exercise, rapid iteration playbook, and user conversation templates.

## Growth & Metrics

Read [references/growth.md](references/growth.md) for metric definitions, targets, and frameworks. Essential metrics:

- **Weekly growth rate:** Target 5-7% WoW (YC benchmark)
- **Retention:** If your retention curve doesn't flatten, fix the product before growing
- **North Star:** Pick ONE metric that best reflects value delivery
- **PMF survey:** Ask users "How disappointed would you be if this product went away?" 40%+ "very disappointed" = strong signal

The growth reference includes metric targets by stage, acquisition channel selection, experiment framework, and retention analysis guidance.

## Fundraising

Read [references/fundraising.md](references/fundraising.md) for the complete fundraise playbook. Key points:

- **Don't fundraise during Weeks 1-4.** Build and talk to users.
- **Build your investor list early** (Weeks 5-6). Tier them: 4-6 dream, 8-12 strong, 15-20 reasonable.
- **Warm intros only.** Use YC partners and batchmates for connections.
- **Momentum is everything.** Run a tight process. Meet investors in parallel.
- **Post-money SAFE, capped.** Use YC's template. Don't over-optimize on terms.

The fundraising reference includes the full timeline, pitch principles, SAFE note guidance, investor outreach strategy, and post-Demo Day management.

## Handling Common Founder Questions

**"Should I pivot?"** — Read [references/building.md](references/building.md) "Product-Market Fit Signals" section. If <40% of users say "very disappointed" without your product, consider pivoting.

**"How fast should we be growing?"** — 5-7% WoW is the YC benchmark. See [references/growth.md](references/growth.md) for targets by stage.

**"When do I start fundraising?"** — Weeks 5-6 for building the list, Weeks 7-8 for meetings, Weeks 9-11 for closing. See [references/fundraising.md](references/fundraising.md).

**"What should I build first?"** — Use the "Worst Viable Product" exercise in [references/building.md](references/building.md). Ship the minimum that tests your core hypothesis.

**"How do I write a pitch?"** — Start with [assets/pitch-deck-template.md](assets/pitch-deck-template.md). Then see fundraising reference for pitch principles.
